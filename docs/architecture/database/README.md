# Database

This directory contains the database design and related resources for SuperPlanify.

Its purpose is to document how the domain model is translated into a relational database, independently of the application implementation.

Contents may include:

* relational database schemas;
* Entity-Relationship Diagrams (ERDs);
* dbdiagram.io definition files;
* SQL scripts;
* database conventions and design notes.

The database design evolves alongside the domain model and reflects the current state of the project.
